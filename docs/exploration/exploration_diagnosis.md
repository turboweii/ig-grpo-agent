# 探索塌缩诊断报告

## 问题定义

在多工具 Agent 的长链路任务中，我们观察到以下三种"探索塌缩"现象：

### 1. 稀疏奖励塌缩

**现象**：Agent 的状态覆盖率在训练早期（~50 steps）达到平台期后停止增长。

**数据**：
- Vanilla GRPO: 状态覆盖率从 5% → 18% → 19%（停滞）
- IG-Full: 状态覆盖率从 5% → 25% → 38% → 45%（持续增长）

**根因**：二元奖励（0/1）且仅在任务结束时给出，长链路中间步骤无反馈。

### 2. 重复模式循环

**现象**：Agent 发现一个可行的工具调用模式后，会不断重复该模式。

**数据**：
- Vanilla GRPO: 28% 的工具调用是 3 轮内的重复
- IG-Full: 12% 的重复率（-57%）

**示例**：
```
Vanilla: search → add → search → add → search → add (循环)
IG-Full:  search → add → get_details → modify → checkout (探索)
```

**根因**：缺乏"已访问"惩罚和工具组合多样性奖励。

### 3. 状态覆盖不足

**现象**：大量 state-tool 组合从未被访问。

**数据**：
- 状态空间大小估计：~1,000,000
- Vanilla GRPO 访问：~180,000 状态（18%）
- IG-Full 访问：~450,000 状态（45%）

**根因**：高频状态过度采样，新状态探索不足。

---

## 解决方案

### JIG 奖励

```python
JIG = α_state × NewStateReward
    + α_tool × NewToolComboReward
    + α_transfer × ToolTransferNovelty
```

| 组件 | 作用 | 权重 |
|------|------|------|
| 状态新颖性 | 鼓励访问新状态 | 0.3 |
| 工具组合新颖性 | 鼓励用新工具处理已知状态 | 0.4 |
| 转移新颖性 | 鼓励罕见的工具序列 | 0.2 |
| 探索持续性 | 防止过早收敛 | 0.1 |

### 课程学习

```python
α(t) = α_0 × (1 - t/T)^β
```

| 阶段 | t/T | α(t) | 行为 |
|------|-----|------|------|
| 早期 | 0-0.3 | 0.8 → 0.6 | 激进探索 |
| 中期 | 0.3-0.7 | 0.6 → 0.3 | 平衡 |
| 晚期 | 0.7-1.0 | 0.3 → 0.1 | 收敛利用 |

### G-Normalization

```python
advantage_normalized = advantage / L^γ
```

其中 γ = 0.5（平方根归一化），防止长轨迹淹 Episode 级信号。

## 消融实验

### 实验 1：IG-Fixed

**配置**：固定 IG 权重 0.5，无课程学习

**结果**：
- Pass@1: 0.175 → 0.195 (+11%)
- 状态覆盖: 18% → 32% (+78%)
- 工具熵: 1.82 → 2.15 (+18%)

**分析**：IG 信号有效促进状态覆盖，但工具多样性提升有限。

### 实验 2：IG-Combo

**配置**：IG-Fixed + 工具组合奖励 + 重复惩罚

**结果**：
- Pass@1: 0.195 → 0.218 (+12%)
- 状态覆盖: 32% → 38% (+19%)
- 工具熵: 2.15 → 2.45 (+14%)
- 冗余率: 22% → 16% (-27%)

**分析**：工具组合奖励显著提升多样性。

### 实验 3：IG-Full

**配置**：IG-Combo + 课程学习 + 探索持续性

**结果**：
- Pass@1: 0.218 → 0.248 (+14%)
- 状态覆盖: 38% → 45% (+18%)
- 工具熵: 2.45 → 2.73 (+11%)
- 泛化 Pass@1: 0.108 → 0.128 (+19%)

**分析**：课程学习确保探索与利用的平衡，探索持续性防止过早收敛。

## 失败案例

### 案例 1：工具名称幻觉

**Vanilla**：
```
用户: 我要买一台笔记本电脑
Agent: 调用 Products.laptop_search (工具不存在)
结果: 错误
```

**IG-Full**：
```
用户: 我要买一台笔记本电脑
Agent: 调用 Products.search (正确工具)
结果: 成功
```

**Vanilla**：
```
Agent: Products.search(category="laptop")
系统: 错误，category 参数应该是 "Computers"
Agent: 重复相同调用（3次）
结果: 失败
```

**IG-Full**：
```
Agent: Products.search(category="laptop")
系统: 错误
Agent: 尝试 Products.get_by_id (探索新工具)
结果: 成功
```

## 结论

1. **探索塌缩是真实问题**：Vanilla GRPO 在多工具场景下覆盖率 < 20%
2. **IG 信号有效**：状态覆盖率提升 78%（IG-Fixed）
3. **工具组合奖励关键**：工具多样性提升 35%（IG-Combo）
4. **课程学习必要**：Pass@1 进一步提升 14%（IG-Full）
5. **完整方案最优**：Pass@1 0.248，覆盖率 45%，工具熵 2.73
